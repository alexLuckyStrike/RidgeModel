# Описание структуры мезоцикла (createMesoCycle)

## 1. Что такое мезоцикл

Мезоцикл — последовательность из 2–4 микроциклов (недель), объединённых общей педагогической направленностью. Каждая неделя мезоцикла = один микроцикл фиксированной длины (sessionsPerWeek).

Связь уровней:
```
Сессия → Микроцикл (неделя) → Мезоцикл (2–4 недели)
```

---

## 2. Типы мезоциклов

| Тип | ID | Назначение | Средний PC1 | Ключевой критерий |
|-----|----|-----------|-------------|-------------------|
| Втягивающий | `recovery_intro` | Вход в тренировочный процесс после перерыва | < −0.5 | Z3+Z4 ≤ 10%, нет avoid |
| Базовый | `base` | Основной тренировочный блок | [−1.0; +0.5] | Z2 ≥ 40%, ≤1 ударной недели |
| Развивающий | `shock` | Максимальная стимуляция | ≥ −0.75 | Z3+Z4 ≥ 15%, ≥1 ударной недели |
| Предсоревновательный | `taper` | Подведение к старту | < −0.5 | Z3+Z4 ≤ 15%, Z1 ≥ 15% |
| Восстановительный | `recovery` | Глубокий отдых | < −1.0 | Z3+Z4 = 0, Z1 ≥ 40% |

Каждый тип × 3 длины (2, 3, 4 недели) × 5 вариантов = **15 вариантов на тип**, итого **75 мезоциклов** на спортсмена (при фиксированном sessionsPerWeek).

---

## 3. Четыре типа динамики

Динамика определяет, как меняется средний PC1 от недели к неделе:

| Динамика | ID | Описание | Пример weeklyPC1 |
|----------|----|---------|-------------------|
| Нарастающая | `ascending` | PC1 растёт каждую неделю | [−2.50, −2.04, −1.58] |
| Убывающая | `descending` | PC1 падает каждую неделю | [−1.58, −2.04, −2.50] |
| Маятниковая | `pendulum` | PC1 меняет направление (↑↓ или ↓↑) | [−1.58, −2.50, −1.58] |
| Ступенчатая | `step` | Плато → скачок (допускает ascending или descending) | [−2.50, −2.04, −1.58, −1.58] |

Распределение динамик по вариантам (для 3 и 4 недель):

| Вариант | 2 недели | 3 недели | 4 недели |
|---------|----------|----------|----------|
| v1 | ascending | ascending | ascending |
| v2 | ascending | pendulum | pendulum |
| v3 | descending | step | step |
| v4 | step | descending | descending |
| v5 | step | ascending/pendulum | ascending/pendulum |

---

## 4. Адаптивный подбор микроциклов (уровни нагрузки)

Ключевая архитектурная особенность: шаблон мезоцикла задаёт не конкретный микроцикл, а **уровень нагрузки** (level). Билдер адаптивно подбирает лучший микроцикл из каталога для данного sessionsPerWeek.

Это решает проблему: один и тот же микроцикл (например, recovery_intro/v1) даёт разный средний PC1 при разном количестве сессий (−1.89 при 3s, −1.58 при 4s, −1.76 при 5s).

### Уровни нагрузки по типам мезоциклов

**Втягивающий мезоцикл:**

| Уровень | Пул микроциклов | Диапазон PC1 | Z3 допустимы | Типичный PC1 |
|---------|----------------|-------------|--------------|-------------|
| lowest | recovery | [−3.0; −1.8] | нет | −2.50 |
| low | recovery, recovery_intro | [−2.2; −1.5] | нет | −2.04 |
| medium | recovery_intro | [−1.8; −1.0] | нет | −1.58 |
| high | recovery_intro | [−1.7; −0.8] | нет | −1.27 |
| bridge | base | [−0.8; −0.3] | нет | −0.66 |

**Базовый мезоцикл:**

| Уровень | Пул микроциклов | Диапазон PC1 | Z3 допустимы | Типичный PC1 |
|---------|----------------|-------------|--------------|-------------|
| unload | recovery_intro | [−2.0; −1.0] | нет | −1.58 |
| base_low | base | [−0.8; −0.3] | нет | −0.66 |
| base_mid | base | [−0.8; 0.0] | да | −0.09 |
| base_high | shock | [−0.2; +0.5] | да | +0.08 |

**Развивающий мезоцикл:**

| Уровень | Пул микроциклов | Диапазон PC1 | Z3 допустимы | Типичный PC1 |
|---------|----------------|-------------|--------------|-------------|
| shock_unload | recovery_intro | [−2.0; −1.0] | нет | −1.58 |
| shock_support | base | [−0.8; −0.3] | нет | −0.66 |
| shock_low | shock | [−0.2; +0.2] | да | −0.14 |
| shock_mid | shock | [−0.2; +0.5] | да | −0.03 |
| shock_high | shock | [−0.1; +0.5] | да | +0.28 |

**Предсоревновательный мезоцикл:**

| Уровень | Пул микроциклов | Диапазон PC1 | Z3 допустимы | Типичный PC1 |
|---------|----------------|-------------|--------------|-------------|
| taper_start | base | [−0.8; −0.3] | нет | −0.66 |
| taper_mid | taper | [−1.7; −0.6] | нет | −1.27 |
| taper_low | taper, recovery_intro | [−2.0; −1.0] | нет | −1.58 |
| taper_finish | recovery, recovery_intro | [−3.0; −1.5] | нет | −2.50 |

**Восстановительный мезоцикл:**

| Уровень | Пул микроциклов | Диапазон PC1 | Z3 допустимы | Типичный PC1 |
|---------|----------------|-------------|--------------|-------------|
| rec_deep | recovery | [−3.0; −2.0] | нет | −2.50 |
| rec_mid | recovery | [−2.5; −1.5] | нет | −2.04 |
| rec_light | recovery, recovery_intro | [−2.2; −1.3] | нет | −1.58 |
| rec_exit | recovery_intro | [−1.8; −1.0] | нет | −1.27 |

### Алгоритм подбора

Для каждого уровня билдер:
1. Собирает кандидатов из пула типов → фильтрует по диапазону PC1 → фильтрует по Z3
2. Исключает уже использованные микроциклы (разнообразие)
3. Сортирует: prefer='high' → максимальный PC1 первым, prefer='low' → минимальный, иначе → ближе к центру диапазона

## 5. Структура объекта мезоцикла

### Верхний уровень (результат createMesoCatalogForAll)

```
[
  {
    athleteId: "ath-1",
    name: "Спортсмен 1",
    catalog: {
      "2": [...],     // 5 вариантов по 2 недели
      "3": [...],     // 5 вариантов по 3 недели
      "4": [...]      // 5 вариантов по 4 недели
    }
  },
  ...                  // 36 спортсменов
]
```

### Вариант мезоцикла (элемент массива)

```
{
  variantId: "v1",                        // Идентификатор (v1–v5)
  name: "Линейный разгон",               // Название
  context: "Восстановительный → ...",    // Когда применять
  type: "recovery_intro",                // Тип мезоцикла (см. §2)
  dynamics: "ascending",                 // Тип динамики (см. §3)
  sessionsPerWeek: 4,                    // Тренировок в неделю (3–6)
  weeks: [...],                          // Массив недель (см. §6)
  metadata: {...},                       // Агрегированные метаданные (см. §7)
  validation: {...}                      // Результат валидации (см. §8)
}
```

## 6. Объект недели (week)

Каждая неделя мезоцикла содержит один полный микроцикл:

```
{
  weekNumber: 1,                          // Номер недели (1-based)
  level: "lowest",                        // Уровень нагрузки (см. §4)
  selectedKey: "recovery/v3",             // Какой микроцикл выбран билдером
                                          //   формат: "{тип_микроцикла}/{вариант}"
  microCycle: {                           // Полный объект микроцикла
    variantId: "v3",
    name: "Глубокое восстановление",
    context: "...",
    type: "recovery",                     // Тип микроцикла (может отличаться от типа мезоцикла!)
    sessions: [...],                      // Массив сессий (см. microcycleDesc.md §5)
    metadata: {...},                      // Метаданные микроцикла
    validation: {...}                     // Валидация микроцикла
  }
}
```

### Важно: тип микроцикла ≠ тип мезоцикла

Втягивающий мезоцикл (`recovery_intro`) может содержать микроциклы типов `recovery`, `recovery_intro` и `base`. Развивающий мезоцикл (`shock`) может содержать `shock`, `base` и `recovery_intro`. Тип мезоцикла определяется общей направленностью, а не типами отдельных недель.

## 7. Объект metadata (мезоцикла)

```
{
  weeklyPC1: [-2.50, -2.04, -1.58],   // Средний PC1 каждой недели (из averagePC1 микроцикла)
  averagePC1: -2.04,                    // Средний PC1 всего мезоцикла
  deltas: [0.46, 0.46],                // Изменения PC1 между неделями
                                        //   deltas[i] = weeklyPC1[i+1] − weeklyPC1[i]
                                        //   Положительная = нагрузка растёт
                                        //   Отрицательная = нагрузка падает
  totalDistribution: {                  // Суммарное распределение зон по ВСЕМ сессиям
    "1": 9,                             //   Всего сессий Зоны 1 во всём мезоцикле
    "2": 3,
    "3": 0,
    "4": 0
  },
  totalRealismProfile: {                // Суммарный профиль реалистичности
    good: 12,
    caution: 0,
    avoid: 0
  },
  totalSessions: 12,                    // Общее количество сессий (недели × sessionsPerWeek)
  weekCount: 3                          // Количество недель
}
```

### Интерпретация weeklyPC1

Это «кривая нагрузки» мезоцикла — главный инструмент визуализации:

```
Неделя 1: PC1 = -2.50  ████████████████████████████████  (глубокий отдых)
Неделя 2: PC1 = -2.04  ██████████████████████████        (лёгкая работа)
Неделя 3: PC1 = -1.58  ████████████████████              (втягивающая)
```

### Интерпретация deltas

Все дельты одного знака → монотонная динамика. Смена знака → маятник. Нули → плато.

```
deltas = [+0.46, +0.46]         → ascending (нарастающая)
deltas = [-0.46, -0.46]         → descending (убывающая)
deltas = [-0.92, +0.92]         → pendulum (маятниковая)
deltas = [+0.46, +0.46, 0]      → step/ascending (ступенчатая)
```

## 8. Объект validation (мезоцикла)

```
{
  valid: true,                          // Прошёл валидацию
  checks: {
    dynamicsValid: true,                // Динамика соответствует заявленной
    dynamicsExpected: "ascending",      // Ожидаемая динамика
    dynamicsActual: "ascending",        // Фактическая динамика
    pc1Check: true,                     // Средний PC1 в допустимом диапазоне
    noAvoid: true,                      // Нет avoid-сессий
    highCheck: true,                    // Z3+Z4 в допустимых пределах
    highZonesRatio: 0                   // Процент Z3+Z4 от общего числа сессий
  },
  warnings: []                          // Пусто = всё ОК
}
```

### Критерии валидации по типам

| Критерий | recovery_intro | base | shock | taper | recovery |
|----------|---------------|------|-------|-------|----------|
| Средний PC1 | < −0.5 | [−1.0; +0.5] | ≥ −0.75 | < −0.5 | < −1.0 |
| Z3+Z4 | ≤ 10% | — | ≥ 15% | ≤ 15% | = 0 |
| Z2 | — | ≥ 40% | — | — | — |
| Z1 | — | — | — | ≥ 15% | ≥ 40% |
| Ударные недели | — | ≤ 1 | ≥ 1 | — | — |
| Нет avoid | ✓ | ✓ | ✓ | ✓ | ✓ |
| Динамика | ✓ | ✓ | ✓ | ✓ | ✓ |

## 9. Пример: чтение мезоцикла recovery_intro/v1 (3 недели, 4 сессии/нед)

```
Тип:       recovery_intro (втягивающий)
Динамика:  ascending (нарастающая)
Длина:     3 недели × 4 сессии = 12 сессий

Неделя 1: recovery/v3  (level=lowest)    PC1=-2.50  [Z1,Z1,Z1,Z1]
Неделя 2: recovery/v1  (level=low)       PC1=-2.04  [Z1,Z1,Z1,Z2]
Неделя 3: rec_intro/v1 (level=medium)    PC1=-1.58  [Z1,Z2,Z1,Z2]

weeklyPC1 = [-2.50, -2.04, -1.58]
deltas    = [+0.46, +0.46]   → нарастающая ✓
averagePC1 = -2.04            → < -0.5 ✓
Z3+Z4     = 0/12 = 0%        → ≤ 10% ✓
avoid     = 0                 → нет ✓
```

Логика: три ступени нарастания. Глубокий отдых → лёгкое включение → полноценная втягивающая работа. Стандартный вход после 2–4 недель перерыва.

## 10. Формула вызова

```javascript
import { createMesoCatalogForAll } from './createMesoCycle'

const sessionsPerWeek = 4

// 5 типов мезоциклов
const mesoRecoveryIntro = createMesoCatalogForAll(allAthletePlans, 'recovery_intro', sessionsPerWeek)
const mesoBase          = createMesoCatalogForAll(allAthletePlans, 'base', sessionsPerWeek)
const mesoShock         = createMesoCatalogForAll(allAthletePlans, 'shock', sessionsPerWeek)
const mesoTaper         = createMesoCatalogForAll(allAthletePlans, 'taper', sessionsPerWeek)
const mesoRecovery      = createMesoCatalogForAll(allAthletePlans, 'recovery', sessionsPerWeek)
```

### Три параметра функции

| # | Параметр | Тип | Описание |
|---|----------|-----|---------|
| 1 | allAthletePlans | Array | Массив из 36 спортсменов (выход computeAllAthletePlans). Каждый содержит baseline и 20 вариантов сессий (5 стратегий × 4 зоны). |
| 2 | type | string | Тип мезоцикла: `'recovery_intro'`, `'base'`, `'shock'`, `'taper'`, `'recovery'` |
| 3 | sessionsPerWeek | number | Тренировок в неделю (3–6). Определяет длину каждого микроцикла внутри мезоцикла. |

### Масштаб

При одном sessionsPerWeek: 36 спортсменов × 5 типов × 15 вариантов = **2 700 мезоциклов**.

При всех sessionsPerWeek (3–6): 36 × 5 × 15 × 4 = **10 800 мезоциклов**.

## 11. Иерархия: от сессии до мезоцикла

```
Мезоцикл (recovery_intro, v1, 3 недели, 4 сессии/нед)
│
├── Неделя 1 (level=lowest → recovery/v3)
│   ├── Сессия 0: Z1, balanced, PC1=-2.50, dV=-16.7%, dP=-16.1%
│   ├── Сессия 1: Z1, variative, PC1=-2.50, dV=-12.5%, dP=-12.4%, dR=+23.6%
│   ├── Сессия 2: Z1, balanced, PC1=-2.50, dV=-16.7%, dP=-16.1%
│   └── Сессия 3: Z1, variative, PC1=-2.50, dV=-12.5%, dP=-12.4%, dR=+23.6%
│
├── Неделя 2 (level=low → recovery/v1)
│   ├── Сессия 0: Z1, balanced, PC1=-2.50
│   ├── Сессия 1: Z1, variative, PC1=-2.50
│   ├── Сессия 2: Z1, balanced, PC1=-2.50
│   └── Сессия 3: Z2, balanced, PC1=-0.66
│
└── Неделя 3 (level=medium → recovery_intro/v1)
    ├── Сессия 0: Z1, balanced, PC1=-2.50
    ├── Сессия 1: Z2, volume_only, PC1=-0.66
    ├── Сессия 2: Z1, variative, PC1=-2.50
    └── Сессия 3: Z2, balanced, PC1=-0.66

Итого: 12 сессий, Z1=9, Z2=3, Z3=0, Z4=0
Средний PC1 = -2.04, динамика ascending
```
