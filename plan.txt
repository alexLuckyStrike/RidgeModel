Было (до доработок):

Стало (после доработок):
docker-compose up --build
├─ nginx:8080       (единая точка входа)
│  ├─ /         → nuxt:3000
│  ├─ /api/*    → backend:3001
│  └─ /cv-api/* → cv-backend:4000   ← НОВОЕ

├─ backend:3001     (основной API)

Конкретные преимущества
1. Единая точка входа (nginx на :8080)
# Было:
localhost:3000   → Nuxt
localhost:3001   → API
localhost:4000   → CV-анализ (отдельно, CORS)
localhost:5173   → Vue dev-сервер (не нужен)

# Стало:
localhost:8080   → ВСЁ (через nginx)
  /         → Nuxt
  /api/*    → Backend
  /cv-api/* → CV-Backend

Плюс: Для клиентов один адрес. Для фронтенда один backendBase.

2. Фронтенд теперь обращается к себе самому (нет CORS)
// Было: прямой вызов (CORS требуется)
fetch('http://localhost:4000/api/analyze', { ... })

// Стало: через свой же BFF (server route)
fetch('/api/cv-analyze', { ... })
  ↓
nuxt/server/api/cv-analyze.post.ts
  ↓
http://cv-backend:4000/api/analyze (внутри Docker-сети)

Плюс:

Нет CORS-проблем
Фронтенд не знает о внутренней архитектуре
Легче менять бэкенды позже
3. Docker Compose = одна команда для всего
# Раньше:
Terminal 1: npm run dev                           (Nuxt + main backend)
Terminal 2: cd mvp-cv-app/backend && npm run dev (CV backend)
Terminal 3: cd mvp-cv-app/vision && python3 ...  (Vision process)
Terminal 4: npm run dev (frontend Vue)            (не нужен, интегрирован в Nuxt)

# Теперь:
docker-compose up --build

ВСЁ запускается одновременно, управляется одним compose-файлом.

Плюс:

Один процесс
Управляется docker
Легче деплоить на сервер
4. Изоляция окружения (контейнер = чистое окружение)
# Раньше: нужно вручную ставить
apt-get install python3 python3-pip tesseract-ocr libgl1 ...
pip install opencv-python numpy pillow pytesseract

# Теперь: Dockerfile делает это один раз
- Каждый разработчик/сервер получает ОДНО И ТО ЖЕ окружение
- Нет "работает у меня, не работает у тебя"

Плюс: Grok-воспроизводимость, меньше проблем с зависимостями.

5. Persistent volumes для данных
cv-backend:

Файлы загруженных полосок и результатов анализа остаются на диске хоста, даже если контейнер перестартует.

6. Правильная маршрутизация с nginx
location /cv-api/ {
    proxy_pass http://cv-backend:4000/api/;
    client_max_body_size 50m;
}

Загрузка больших файлов (до 50МБ) работает надёжнее

Session ID сохраняется через X-Session-ID

  ↓
Пробрасывает запрос на: http://localhost:4000/api/analyze
  ↓
CV-бэкенд обрабатывает, запускает Python
  ↓
Возвращает JSON с результатами
  ↓
Компонент получает результаты и отображает их в planner
