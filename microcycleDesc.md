# Описание структуры микроцикла (createMicroCycle)

## 1. Зоны функционального состояния

Зона — атрибут **одной тренировочной сессии**. Определяется значением интегрального индекса PC1, рассчитанного по 4 биохимическим маркерам мочи (креатинин, белок, миоглобин, кетоновые тела).

| Зона | Название | Граница PC1 | Центр PC1 | Биохимический смысл |
|------|----------|-------------|-----------|---------------------|
| 1 | Норма | PC1 < −1.826 | −2.50 | Все маркеры в норме. Восстановительные процессы преобладают. Минимальный стресс-ответ. |
| 2 | Умеренный отклик | −1.826 ≤ PC1 < +0.497 | −0.66 | Превышен порог attention (креатинин). Адаптационная реакция. Рабочая зона тренировочного процесса. |
| 3 | Выраженный отклик | +0.497 ≤ PC1 < +1.575 | +1.04 | Превышен порог exceeding (миоглобин). Микроповреждения мышц. Развивающие тренировки. |
| 4 | Критический отклик | PC1 ≥ +1.575 | +1.70 | Превышен порог danger (креатинин). Экстремальный стресс. Допустимо кратковременно с обязательным восстановлением. |

## 2. Стратегии обратной задачи

Стратегия определяет **каким параметром нагрузки** достигается целевая зона. Уравнение модели:

```
ΔPC1 = β_V·dV + β_P·dP + β_R·dR
```

где β_V = 7.48, β_P = 7.79, β_R = −2.54 — коэффициенты ridge-регрессии.

5 стратегий — 5 разных способов решить одно уравнение с тремя неизвестными:

| Стратегия | ID | Что меняется | Ограничения | Педагогический смысл |
|-----------|----|-------------|-------------|---------------------|
| Только объём | `volume_only` | dV | dP=0, dR=0 | Базовые/объёмные тренировки. Школа Шейко. |
| Только интенсивность | `pace_only` | dP | dV=0, dR=0 | Скоростно-силовые тренировки. Метод Симмонса. |
| Только восстановление | `recovery_only` | dR | dV=0, dP=0 | Управление через отдых. **Исключена для Зоны 1** (avoid). |
| Сбалансированная | `balanced` | dV, dP | dR=0, dV/dP пропорциональны β | Равномерное изменение объёма и интенсивности. |
| Вариативная | `variative` | dV, dP, dR | Все три пропорциональны β и σ | Мультипараметрический стимул. Учитывает вариативность по σ. |

### Связь стратегий с зонами (профиль реалистичности)

Не все стратегии реалистичны для всех зон. Проверка `checkRealism` даёт статус:

| Стратегия | Зона 1 | Зона 2 | Зона 3 | Зона 4 |
|-----------|--------|--------|--------|--------|
| volume_only | caution | good | good | good |
| pace_only | caution | good | good | caution |
| recovery_only | **avoid** | good | good | caution |
| balanced | good | good | good | good |
| variative | good | good | good | good |

- **good** — дельты в пределах ±2σ от обучающих данных
- **caution** — дельты в пределах ±3σ, но выходят за ±2σ
- **avoid** — дельты за пределами ±3σ (recovery_only для Зоны 1: R +98.4%, 3.7σ)

## 3. Структура объекта микроцикла

### Верхний уровень

```
{
  athleteId: "ath-1",        // Идентификатор спортсмена
  name: "Спортсмен 1",      // Имя спортсмена
  catalog: {                 // Каталог вариантов, ключи — длина микроцикла
    "3": [...],              // 5 вариантов по 3 сессии
    "4": [...],              // 5 вариантов по 4 сессии
    "5": [...],              // 5 вариантов по 5 сессий
    "6": [...]               // 5 вариантов по 6 сессий
  }
}
```

### Вариант микроцикла (элемент массива)

```
{
  variantId: "v1",                    // Идентификатор варианта (v1–v5)
  name: "Объёмный шок минимальный",  // Название варианта (русское)
  context: "...",                     // Педагогическое описание: когда применять
  type: "shock",                     // Тип микроцикла (см. §4)
  sessions: [...],                   // Массив тренировочных сессий
  metadata: {...},                   // Агрегированные метаданные (см. §6)
  validation: {...}                  // Результат валидации (см. §7)
}
```

## 4. Типы микроциклов

| Тип | ID | Зоны | Назначение | Ключевые критерии |
|-----|----|------|-----------|-------------------|
| Восстановительный | `recovery` | 1, 2 | Глубокий отдых после нагрузки | Z1 ≥ 50%, нет Z3/Z4, PC1 < −1.0 |
| Втягивающий | `recovery_intro` | 1, 2 | Вход в тренировочный процесс | Z1+Z2 ≥ 90%, нет Z3/Z4, PC1 < −1.0 |
| Базовый | `base` | 1, 2, 3 | Основная работа | Z2 ≥ 50%, ≤1 Z3, нет Z4, PC1 ∈ [−0.7; +0.5] |
| Ударный | `shock` | 1, 2, 3, 4 | Развивающий стресс | ≥1 Z3/Z4, ≥1 Z1 компенсаторная, PC1 ≥ −0.15 |
| Подводящий | `taper` | 1, 2, 3 | Подведение к старту | Спирмен ≤ −0.5, lastZone=1, нет Z4, PC1 < 0 |

Каждый тип × 4 длины (3–6 сессий) × 5 вариантов = **20 вариантов на тип**, итого **100 микроциклов** на спортсмена.

## 5. Объект сессии (session)

Одна тренировочная сессия внутри микроцикла:

```
{
  sessionIndex: 0,           // Порядковый номер сессии (0-based)
  zone: 4,                   // Целевая зона функционального состояния (1–4)
  strategy: "volume_only",   // Стратегия обратной задачи (см. §2)
  predictedPC1: 1.7,         // Предсказанное значение PC1 (центр зоны)
  realism: "good",           // Статус реалистичности: good / caution / avoid
  deltas: {                  // Относительные отклонения от baseline спортсмена
    dV: 0.2273,              //   Объём: +22.7% от индивидуальной нормы
    dP: 0,                   //   Интенсивность: без изменений
    dR: 0                    //   Восстановление: без изменений
  },
  description: "V +22.7%"   // Человекочитаемое описание отклонений
}
```

### Что означают deltas

Это **относительные отклонения** от индивидуального baseline спортсмена:

```
V_сессии = V_baseline × (1 + dV)    // Абсолютный объём тренировки
P_сессии = P_baseline × (1 + dP)    // Абсолютная интенсивность
R_сессии = R_baseline × (1 + dR)    // Абсолютное восстановление
```

Например, для ath-1 с baseline V=8426.875, P=75.375, R=3.706:
- dV = +0.2273 → V = 8426.875 × 1.2273 = 10342 кг (объём)
- dV = −0.1671 → V = 8426.875 × 0.8329 = 7018 кг (сниженный)

## 6. Объект metadata

Агрегированные характеристики микроцикла:

```
{
  distribution: {            // Распределение сессий по зонам
    "1": 1,                  //   Количество сессий Зоны 1
    "2": 0,                  //   Количество сессий Зоны 2
    "3": 1,                  //   Количество сессий Зоны 3
    "4": 1                   //   Количество сессий Зоны 4
  },
  averagePC1: 0.08,          // Средний PC1 по всем сессиям микроцикла
  spearman: -0.5,            // Корреляция Спирмена (sessionIndex vs PC1)
                             //   Отрицательная = нагрузка убывает
                             //   Положительная = нагрузка нарастает
  realismProfile: {          // Профиль реалистичности всех сессий
    good: 3,                 //   Количество good-сессий
    caution: 0,              //   Количество caution-сессий
    avoid: 0                 //   Количество avoid-сессий (запрещены)
  },
  dominantStrategy: "volume_only",  // Наиболее частая стратегия
  firstZone: 4,              // Зона первой сессии
  lastZone: 3                // Зона последней сессии
}
```

### Интерпретация averagePC1

Средний PC1 микроцикла определяет его «тяжесть»:

| Диапазон PC1 | Характер микроцикла |
|-------------|---------------------|
| −2.50 … −1.80 | Глубокий отдых (recovery) |
| −1.80 … −1.00 | Лёгкая работа (recovery_intro) |
| −1.00 … −0.30 | Базовая работа (base) |
| −0.30 … +0.50 | Ударная работа (shock) |

### Интерпретация spearman

Коэффициент Спирмена показывает направленность динамики внутри микроцикла:

| Значение | Интерпретация |
|----------|---------------|
| ≤ −0.5 | Убывающая (подводящий микроцикл) |
| −0.5 … +0.5 | Волнообразная / равномерная |
| ≥ +0.5 | Нарастающая |

## 7. Объект validation

Результат проверки микроцикла на соответствие критериям типа:

```
{
  valid: true,               // Прошёл ли валидацию (true/false)
  checks: {                  // Детали по каждому критерию (зависят от типа)
    highCheck: true,         //   Пример для shock: есть Z3/Z4
    highZones: 2,            //   Количество Z3+Z4 сессий
    compensationCheck: true, //   Есть Z1 компенсаторная
    zone1Count: 1,           //   Количество Z1 сессий
    pc1Check: true,          //   Средний PC1 в допустимом диапазоне
    noAvoid: true            //   Нет avoid-сессий
  },
  warnings: []               // Массив строк-предупреждений (пусто = всё ОК)
}
```

## 8. Пример: чтение shock-микроцикла v1 при 3 сессиях

```
Тип: shock (ударный)
Длина: 3 сессии

Сессия 0: Зона 4 (volume_only) → dV=+22.7%, PC1=+1.70 [good]
Сессия 1: Зона 1 (balanced)    → dV=−16.7%, dP=−16.1%, PC1=−2.50 [good]
Сессия 2: Зона 3 (volume_only) → dV=+13.9%, PC1=+1.04 [good]

Средний PC1 = (+1.70 − 2.50 + 1.04) / 3 = +0.08
Spearman = −0.5 (нагрузка убывает)
Зоны: Z1=1, Z3=1, Z4=1
Реализм: good=3, caution=0, avoid=0
```

Логика: максимальный ударный пик (Z4) → компенсаторная (Z1) → развивающая (Z3). Классическая схема «удар—восстановление—закрепление».

## 9. Формула вызова

```javascript
import { createCatalogForAll } from './createMicroCycle'

// 5 типов × 20 вариантов = 100 микроциклов на спортсмена
const shockCatalogs     = createCatalogForAll(allAthletePlans, 'shock')
const recoveryIntroCatalogs = createCatalogForAll(allAthletePlans, 'recovery_intro')
const baseCatalogs      = createCatalogForAll(allAthletePlans, 'base')
const taperCatalogs     = createCatalogForAll(allAthletePlans, 'taper')
const recoveryCatalogs  = createCatalogForAll(allAthletePlans, 'recovery')
```

Масштаб: 36 спортсменов × 100 микроциклов = **3 600 микроциклов**.
